目标平台微信小程序（适配 iOS / Android）

二、游戏类型评估

2.1 市场定位

微信小游戏生态中，当前主流品类分布：

品类代表产品竞争度适合度
消除/合成开心消消乐、合成大西瓜极高⭐⭐
模拟经营动物餐厅、乌冬的旅店高⭐⭐⭐
益智解谜羊了个羊、Brain Test高⭐⭐⭐
Roguelike 卡牌月圆之夜、小丑牌（Balatro）中等⭐⭐⭐⭐⭐
竞技/IO贪吃蛇大作战高⭐⭐

2.2 选型理由

Roguelike + 卡牌构筑 这个品类经多年验证（杀戮尖塔、月圆之夜、Balatro），具备以下优势：

  1. 高复玩性 — 每次开局卡组/地图/敌人随机组合，理论上无限可重玩
  2. 短视频友好 — 一局 10-15 分钟，天然适配碎片化时间
  3. 社交裂变空间大 — 牌组分享、每日挑战排名、好友对战
  4. 商业化成熟 — 角色解锁、皮肤、卡牌扩展包、体力制度
  5. 美术成本可控 — 卡牌+棋盘，不需要复杂场景建模
  6. 微信小游戏尚未饱和 — 对比消除/合成赛道，Steam 移植向 Roguelike 卡牌在微信生态中竞争度较低

2.3 差异化亮点

竞争产品特点我们的差异化
月圆之夜单机卡牌，剧情向棋盘走位 + 卡牌，增加位置策略维度
小丑牌(Balatro)扑克牌+德扑水墨国风，中国传统文化题材
杀戮尖塔爬塔线性路线棋盘点阵，每回合在 5x5 棋盘上走位

三、核心玩法设计

3.1 核心循环

开始对局 → 抽手牌(4张) → 移动+出牌 → 敌人行动 → 结算回合 → 选牌/商店 → 进入下一层

3.2 战斗系统

棋盘：5×5 九宫格棋盘，玩家棋子 + 敌方棋子 + 障碍物/宝箱

行动点：每回合 3 点行动力，可用于：

  • 移动（消耗 1 点）：向相邻格子走一步
  • 出牌（消耗卡面标注费用）：打出卡牌触发效果
  • 待机：保留行动力（上限 5 点）

胜利条件：击败本层所有敌方棋子 → 进入下一层

失败条件：玩家 HP 归零 → 结算本次奖励

3.3 卡牌体系

类型费用效果说明稀有度
攻击1-3对相邻/直线/全屏敌人造成伤害普通/稀有/史诗
防御1-2获得护盾、闪避、减伤普通/稀有
位移1-2瞬移、换位、击退敌人普通/稀有
策略1-4抽牌、复制、强化、冰冻敌人稀有/史诗
绝技3-6全屏大招、召唤、变身史诗/传说

3.4 流派构筑

卡牌分为三大流派，每局可混合但鼓励专精：

流派核心机制代表牌通关风格
🗡️ 墨剑流连击累加、暴击倍率连斩(+1攻击次数)、剑气(远程)、破甲高爆发速杀
🛡️ 墨盾流护盾反伤、叠甲成长铁壁(+3盾)、荆棘(反弹30%)、嘲讽防守反击
🌀 墨法流元素叠加、连锁反应火球(灼烧)、冰锥(冻结)、雷击(连锁)控场+AOE

3.5 成长系统

单局内成长（Roguelike 局内构筑）：

  • 每击败一层 BOSS → 三选一 「墨宝」（永久增益）
  • 商店：金币购买卡牌/移除卡牌/回复 HP
  • 事件：随机遭遇（宝箱、陷阱、NPC 交易）

局外成长（长期留存）：

  • 角色解锁：初始 1 角色，通关解锁新角色（每个角色有专属初始牌）
  • 墨宝图鉴：收集所有墨宝，获得全局属性加成
  • 段位系统：青铜 → 白银 → 黄金 → 墨玉 → 传说
  • 每日挑战：固定种子，所有人同一局面竞争排名

四、数值策划

4.1 基础属性公式

属性基础值成长公式说明
生命值(HP)60baseHP + 10 × 楼层数每过一层+10 上限
攻击力(ATK)8baseATK + cardBonus基础+卡牌加成
护甲(ARM)0armCards × stack回合结束时清空
行动力(AP)33 + relicBonus每回合固定恢复

4.2 伤害公式

最终伤害 = 攻击力 × 技能倍率 × (1 + 暴击加成) × 属性克制 − 护甲

暴击率基础 10%，每点「灵韵」属性 +2% 暴击 · 暴击伤害默认 150%

4.4 经济系统

来源产出周期
战斗胜利10-50 金币每局约 80-150 金币
每日签到50 金币 + 随机卡牌每日 1 次
段位奖励100-500 金币 + 限定皮肤每月结算
看广告双倍金币 / 复活 / 额外抽牌每日限 5 次
付费购买648 墨玉 = ¥648（首充双倍）按需

设计原则：免费玩家通过每日任务+看广告能满足核心体验，付费玩家加速收集和外观解锁。

4.5 卡牌平衡原则

  • 费用效率：1 费卡牌提供 1.0x 价值，2 费 2.2x，3 费 3.5x（鼓励高费牌）
  • 稀有度缩放：普通→稀有→史诗→传说，强度倍率 1.0 : 1.5 : 2.2 : 3.5
  • 流派平衡：墨剑(速攻) vs 墨盾(防守) vs 墨法(控场)，理论通关率差距 < 5%
  • 卡池稀释机制：卡组 ≥ 20 张后，商城刷出核心牌概率动态提升

六、技术栈

6.1 技术架构

层级选型理由
游戏引擎Cocos Creator 3.x / Phaser 3Cocos 微信小游戏支持最好，官方有完整发布流程；Phaser 更轻量适合小团队
编程语言TypeScript类型安全，开发效率高，Cocos/Phaser 均原生支持
UI 框架引擎内置 UI + 自定义组件卡牌 UI、棋盘渲染、动画系统独立开发
本地存储微信小游戏 LocalStorage + IndexedDB存档/设置/卡牌收集数据持久化
后端服务微信云开发 / LeanCloud排名、每日挑战、用户数据同步
资源 CDN微信 CDN / 腾讯云 COS卡牌美术资源按需加载
CI/CDGitHub Actions → 自动发布体验版同 AutoQuantitize 模式

6.2 推荐方案：Phaser 3 + TypeScript

考虑到这是一个 2D 卡牌棋盘游戏，不需要 3D 渲染，推荐 Phaser 3：

  • 包体 ~300KB（gzip），远小于 Cocos 的 ~5MB
  • 微信小游戏的包体限制是 首包 4MB + 分包 2MB×4，Phaser 留出大量空间给美术资源
  • 纯 TypeScript，无引擎编辑器依赖，全代码驱动
  • 社区有现成的微信小游戏适配插件
  • 与 AutoQuantitize 同一技术栈（Node/TS），降低维护成本

八、测试方案

8.1 测试层次

层级测试内容工具/方法频率
单元测试战斗结算、伤害公式、卡牌效果、AI 逻辑Jest / Vitest，Node 环境直接运行每次代码提交
集成测试回合流程、牌组管理、状态叠加Phaser 模拟环境 + 自定义 TestScene每日
UI 测试棋盘渲染、卡牌拖放、动画微信开发者工具预览 + 真机测试每周
平衡性测试各流派通关率、卡牌使用率、经济循环自动化模拟对局（Python script）每次数值调整
性能测试包体大小、FPS、内存、启动速度微信开发者工具 Performance 面板每周
小规模封测用户体验、留存数据、付费转化微信扫码邀请 50-200 人MVP 完成后

8.2 自动化测试策略

平衡性自动化测试（关键）：

  • 编写 Python 脚本，模拟 1000 局 AI vs AI 对局
  • 统计各流派通关率 → 目标差值 < 5%
  • 统计单张卡牌选取率 / 胜率 → 识别过强/过弱卡牌
  • 统计平均通关回合数 / HP 消耗 → 验证难度曲线
  • 每次数值改动自动跑一次 → CI 卡点

回归测试：每次出新卡牌/新角色，跑全量模拟对局，确保不破坏现有平衡。

8.3 微信审核注意事项

  • 首包 ≤ 4MB，分包总大小 ≤ 8MB（Phaser 3 ~300KB，主要为美术资源）
  • 必须有隐私政策（游戏内设置页面）
  • 必须有用户协议（参照 StockMonitor 的 terms 页面）
  • 棋牌类游戏不属于「赌博」范畴，但避免「赌」「博」「彩」等关键词
  • 支付需接入微信支付（小游戏虚拟支付）
  • 广告接入微信广告组件（激励视频 + Banner）

九、风险与应对

风险概率影响应对方案
卡牌平衡难调高中自动化模拟测试 + 快速热更新（每周平衡补丁）
美术资源不足中中先用占位图开发核心玩法，配合 AI 生成美术进行替换
微信审核驳回中高提前准备合规材料，参考 StockMonitor 过审经验
用户留存不达预期中高MVP 阶段即上线数据分析，快速调整留存策略
竞品抄袭低低核心竞争力在数值平衡和持续运营，非单一玩法

2.4 后处理流程

后处理管线 (Python脚本)
1. 角色表裁切 → sprite_selector.py (10维评分, 凸包mask, 自动选最佳帧)
2. 去背景 → rembg (GPU加速, 输出RGBA透明PNG)
3. 缩放 → Pillow resize 至 128x128px (Lanczos插值)
4. 精灵表打包 → 拼合为单张精灵表PNG (动作x方向)
5. 验证 → 自动检测边缘纯净度(>95%透明), 角色完整性

8.2 工作流选择决策树

需要生成素材? | +-- 是角色精灵? | +-- 多pose参考 --> chibi_character_sheet.json | +-- 精确单帧动作 --> chibi_pose_control.json (+ ControlNet) | +-- 连续动作序列 --> anim_cn_16poses.json (AnimateDiff) | +-- 是场景背景? --> chibi_ink_txt2img.json (高分辨率) +-- 是卡牌插画? --> chibi_ink_txt2img.json (240x320) +-- 是敌人精灵? --> chibi_character_sheet.json (调整prompt) +-- 是UI元素? | +-- 按钮/面板 --> txt2img 或 代码绘制 | +-- 进度条/格子 --> 代码绘制 (Phaser Graphics) +-- 是道具/特效? +-- 静态 --> txt2img (小尺寸) +-- 动画 --> AnimateDiff 或 代码粒子

九、质量验收标准

9.1 角色精灵

验收项标准检测方法
背景纯净度边缘像素 >95% 为纯白/透明rembg去背后自动检测
角色完整性全身可见, 宽高比 0.4-1.2sprite_selector.py 凸包检测
风格一致性HSV直方图相关 >0.85Python HSV对比脚本
动作可读性评分 >75分 (10维评分)sprite_selector.py 自动评分
边缘清晰度Sobel梯度 >阈值自动检测
文字/伪影无可见文字/水印顶部水平结构检测

9.2 场景背景

验收项标准
尺寸800x600
图层分离各层独立, 可单独视差滚动
风格一致3章色调统一
无伪影无接缝/拉伸/重复纹理

验收项标准
尺寸240x320 px
构图主体居中, 四周留白

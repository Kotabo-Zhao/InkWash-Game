## 一、美术风格定位

### 水墨风格

**参考**：中国传统水墨画 + 现代UI设计

**视觉关键词**：
- 宣纸质感（淡黄/米色底）
- 墨色浓淡（多层次灰度）
- 朱砂点缀（红色高光）
- 书法笔意（手写体标题）
- 印章装饰（方形红印）

**纯代码绘制**（Phaser Graphics API）：
- 矩形/圆形几何图形
- 颜色填充和描边
- 文字渲染（STKaiti字体）
- 粒子效果（墨滴、光晕）

## 六、分层控制策略（v2核心）

> 纯靠文字Prompt无法保证角色一致性和动作连贯性。必须用分层控制。

### 6.1 问题与方案

| 问题 | 根因 | 解决方案 | 对应工具 |
|------|------|---------|---------|
| 角色不一致 | 每次生成独立seed，外观飘移 | IP-Adapter + 角色参考图锁定外观 | ComfyUI IP-Adapter |
| 动作帧不连贯 | 纯文本无法精确控制姿态 | 参考动作图 → VNCCS POSE STUDIO 转换为角色姿势 → ControlNet 驱动 | VNCCS POSE STUDIO + ComfyUI ControlNet |
| 风格飘移 | 仅靠prompt不够稳定 | 风格LoRA + 固定negative | ComfyUI LoRA |
| 无法复现 | 没记录参数 | 版本管理器记录所有参数 + git代码版本控制 | scripts/art_pipeline.py + git |

### 6.2 生成流程（四步法）

```
第一步：角色设计（一次性）
  └─ ComfyUI生成角色参考图（正面/侧面/背面三视图）
  └─ 人工审核 → 锁定角色外观
  └─ 保存到 assets/refs/{角色}_ref.png

第二步：动作姿态采集
  └─ 从免费素材网站下载参考动作帧
  └─ VNCCS POSE STUDIO 提取骨骼姿态
  └─ 保存为 OpenPose 格式：assets/refs/pose/{action}_{direction}_{frame}.png

第三步：批量生成（自动化）
  └─ 加载角色参考图 → IP-Adapter 锁定外观
  └─ 加载姿态图 → ControlNet 控制姿态
  └─ 风格LoRA → 锁定水墨风格
  └─ 版本管理器 → 记录所有参数

第四步：后处理
  └─ 统一尺寸（128×128）
  └─ 去背景（透明PNG）
  └─ 打包精灵表
```

### 6.3 角色参考图生成

**工作流说明**：
- 使用 SDXL Base 1.0 作为基础模型
- 生成角色三视图（正面、侧面、背面）
- 固定 seed=42 确保可复现
- 512×512 输出，后续缩放到128×128

**Prompt模板**：
```
masterpiece, best quality, ink wash painting style, traditional chinese art,
character design sheet, {角色中文名}, {角色描述},
multiple views, front side back,
white background, clean lines, game asset
```

**审核标准**：
- [ ] 角色外观符合世界观描述
- [ ] 三视图角色一致（衣服、发型、体型）
- [ ] 线条清晰，适合缩小到128×128
- [ ] 水墨风格明显

### 6.4 动作姿态控制（VNCCS POSE STUDIO方案）

> **VNCCS POSE STUDIO** 是 ComfyUI 自定义节点（免费开源），提供交互式3D姿态编辑环境。
> 官方仓库：`AHEKOT/ComfyUI_VNCCS_Utils`
> 安装方式：ComfyUI Manager 搜索 "VNCCS" 安装 `ComfyUI_VNCCS_Utils`

**工具定位**：
- 它是一个 **ComfyUI节点**，不是独立软件
- 核心能力：在ComfyUI工作流内创建精确的3D姿态控制图像
- 输出：pose/control images（姿态控制图），喂给 ControlNet

**三种姿态输入方式**：
1. **手动摆姿** — 在节点内的3D视口中直接拖拽骨骼关节
2. **SAM 3D Body 自动检测** — 导入参考动作图片 → 自动检测姿态 → 应用到3D人偶
3. **Mixamo FBX 导入** — 直接导入Mixamo下载的3D动画帧（FBX格式）

**为什么用 VNCCS POSE STUDIO？**
- 纯文本prompt无法精确控制姿态 → 每次生成姿势都不同
- VNCCS输出精确的3D姿态控制图 → ControlNet可以精确还原
- 支持从参考图自动检测姿态（SAM 3D Body）→ 无需手摆每一帧
- 多姿态Tab + 姿态库 → 可复用、可批量

**姿态参考来源**（免费可商用）：
- Mixamo（mixamo.com）— 3D动画，下载FBX直接导入VNCCS
- OpenGameArt.org — 2D游戏素材截图
- itch.io/game-assets — 免费游戏资源
- Kenney.nl — CC0游戏素材

**操作流程**：
```
第一步：准备角色参考图
  └─ ComfyUI生成角色三视图（正面/侧面/背面）
  └─ 保存到 assets/refs/{角色}_ref.png

第二步：VNCCS POSE STUDIO 创建姿态控制图
  ├─ 方式A（推荐）：Mixamo下载FBX动画 → 导入VNCCS → 逐帧导出姿态图
  ├─ 方式B：参考动作图 → SAM 3D Body自动检测 → 微调 → 导出姿态图
  └─ 方式C：手动在3D视口摆姿 → 导出姿态图
  
第三步：ComfyUI工作流批量生成
  ├─ 角色参考图 → IP-Adapter（锁定外观）
  ├─ VNCCS姿态图 → ControlNet OpenPose（控制姿势）
  ├─ 风格LoRA → 锁定水墨风格
  └─ 输出最终精灵帧

第四步：后处理
  └─ 统一尺寸（128×128）→ 去背景 → 打包精灵表
```

**输出命名**：
```
assets/refs/pose/{角色}_{方向}_{动作}_{帧号}.png
例：assets/refs/pose/protagonist_e_walk_0.png
```

**VNCCS关键参数**：
| 参数 | 说明 |
|------|------|
| body sliders | 年龄、性别混合、体重、肌肉、身高 — 匹配角色体型 |
| camera | 输出尺寸、缩放、偏移 — 控制构图 |
| export mode | LIST（批量单图）/ GRID（合并图） |
| lighting | 环境光、方向光、点光 — 可生成灯光prompt |
| pose tabs | 多个独立姿态状态 — 用于批量输出 |

### 6.5 风格约束（风格LoRA）

**训练目标**：
- 基于水墨画训练数据
- 触发词：`ink wash style`
- 确保所有素材风格统一

**如果没有自定义LoRA**，可用IP-Adapter + 风格参考图替代：
- 准备一张标准水墨风格参考图
- 通过IP-Adapter注入风格
- 所有生成任务使用同一张风格参考图

## 七、版本管理

### 7.1 为什么需要版本管理

问题场景：
- 上周生成的主角很好看，这周重新生成完全变了样
- 想修改某个角色，但忘了用的什么模型和参数
- 不同人生成出来的素材风格完全不一样

### 7.2 版本记录内容

每次生成自动保存到 `assets/versions/{version_id}.json`：
```json
{
  "version_id": "character_protagonist_1721361600",
  "timestamp": 1721361600,
  "asset_type": "character",
  "asset_id": "protagonist",
  "params": {
    "character": { ... },
    "total_frames": 120,
    "ref_image": "protagonist_ref.png"
  },
  "model_version": {
    "base_model": "sd_xl_base_1.0.safetensors",
    "style_lora": "ink_wash_style_v1.safetensors",
    "controlnet": "control_v11p_sd15_openpose.pth",
    "ip_adapter": "ip-adapter-plus_sd15.safetensors",
    "version_hash": "a3f2b1c4"
  },
  "style_config": {
    "steps": 30,
    "cfg_scale": 7.5,
    "sampler": "dpm_2m_karras"
  }
}
```

### 7.3 版本命令

```bash
# 列出版本
python scripts/art_pipeline.py --versions

# 使用特定版本参数重新生成
python scripts/art_pipeline.py --character protagonist --version character_protagonist_1721361600
```

## 十、性能优化建议

### 精灵表优化
- 合并小图为纹理图集
- 使用TexturePacker打包
- 控制单张精灵表大小（< 2048×2048）

### 背景优化
- 视差滚动只更新可见层
- 远处背景降低分辨率
- 预渲染静态层

### 特效优化
- 粒子使用对象池
- 控制每帧特效数量
- 低性能设备减少粒子数
